Bisect

—verb: to debug using the process of elimination.

Bisect allows users to test program state against expected values. Tests results are used as bisection points to narrow-down towards the root-cause of the underlying issue.

Module owners employ their grasp of system architecture to construct a decision tree by arranging bisection point that first test for anomalous behavior at the system level and later narrow down to module level root-causes.

Customer engineering, Software Test Teams and New developers can integrate bisection trees with continuous automation to quickly classify and narrow-down the source of the problem and gain better system understanding in the process.
