Bisect

—verb: to debug using the process of elimination.

Bisect is a tool that allows the functional state of programs to be tested against expected results. Tests results are used as bisection points to eliminate and narrow-down towards the root-cause of the underlying issue

Module owners construct bisection trees by checking for signs for failure at the system level and creating bisection points that lead towards module level root-causes.

All users leverage the bisection tree to help narrow-down to the root-cause of the problem
